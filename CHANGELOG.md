- **protocol**: Add DIR prefix byte and fix serial config for DPS-150 communication
  ([`d748d0b`](https://github.com/Philipp-IoT/fnirsi_protocol/commit/d748d0b7fc048f96a823174323f209730872b14b))

BREAKING CHANGE: Wire protocol now includes 0xf1/0xf0 direction prefix byte. DeviceStatus fields
  renamed from millivolt/milliamp ints to float V/A.

Confirmed from Windows USBPcap capture and live hardware test (2026-03-29):

Protocol fixes: - Prepend 0xf1 (TX) direction byte to every outgoing frame - Consume 0xf0 (RX)
  direction byte from every incoming frame - Baud rate 115200 → 9600 (from CDC SET_LINE_CODING
  analysis) - Serial line state: DTR=off, RTS=on (from SET_CONTROL_LINE_STATE) - Add start-session
  magic (b0 00 01 01 01) after READY handshake - Add connect/disconnect handshake in DPS150 context
  manager - Add read_frame() with length-delimited framing (no stop byte)

Code changes: - protocol.py: add DIR_TX/DIR_RX constants, START_SESSION_MAGIC - connection.py:
  write() prepends 0xf1, read_frame() strips 0xf0 - device.py: full connect/disconnect handshake,
  float V/A API - cli.py: update defaults to 9600 baud, float status fields

Documentation: - All protocol docs updated with DIR prefix in wire format - Capture annotations
  corrected (USBPcap, not usbmon artefacts) - Kaitai .ksy updated with DIR byte and
  start_session_magic enum - RE methodology updated for USBPcap workflow - CMD 0xb0 resolved as
  start-session magic in known issues
